# 🐹 togglemesh-go

[![GitHub Repo](https://img.shields.io/badge/GitHub-ToggleMesh-blue?logo=github)](https://github.com/sdwck/ToggleMesh)

Official, high-performance Go Server SDK for **ToggleMesh** — the real-time feature flag and configuration management engine.

This SDK is designed for **Server-Side environments**, leveraging secure **Local Evaluation** through Server-Sent Events (SSE) to evaluate targeting rules directly in memory with zero network latency.

## ✨ Features

*   **Zero-Latency Evaluation:** All targeting rules are evaluated locally in memory. `client.IsEnabled()` never makes a network request.
*   **Real-time SSE Synchronization:** Flag updates are instantly pushed from the server to your Go application.
*   **Analytics Event Tracking:** Send custom telemetry and experiment conversions seamlessly:
    ```go
    client.TrackEvent("video_played", 1.0, "user_123", context)
    ```
*   **Native Go Code Generation CLI:** Generate strongly-typed Go constants from your server flags directly via `go generate`.

---

## 📦 Installation

Install the ToggleMesh Go SDK via `go get`:

```bash
go get github.com/sdwck/ToggleMesh/sdks/go@latest
```

## 🛠️ Code Generation CLI

ToggleMesh provides a native Go CLI to generate feature flag constants, eliminating hardcoded strings and typos.

### 1. Install the CLI globally
```bash
go install github.com/sdwck/ToggleMesh/sdks/go/cmd/togglemesh@latest
```

### 2. Configure Authentication (One-time)
Securely save your Personal Access Token (PAT) and Project ID:
```bash
togglemesh config -key tmp_your_pat_token -project your-project-id -address https://api.togglemesh.dev
```

### 3. Generate Flags automatically
Add a `//go:generate` directive to any Go file in your project:
```go
//go:generate togglemesh sync -out flags.go -pkg main
package main
```

Run standard Go generation:
```bash
go generate ./...
```
This produces a `flags.go` file with all your feature flags exposed as Go constants in `PascalCase`.

## 🚀 Quick Start

### 1. Initialize the Client
Initialize the `ToggleMeshClient` as a singleton when your application starts. You must use a **Server API Key** for server-side evaluation.

```go
package main

import (
	"fmt"
	"time"

	"github.com/sdwck/ToggleMesh/sdks/go/togglemesh"
)

func main() {
	options := &togglemesh.ToggleMeshOptions{
		BaseURL: "https://api.togglemesh.dev",
		APIKey:  "tm_server_your_api_key_here", // Requires Server Key
		// Optional logger
		// Logger: slog.Default(), 
	}

	client, err := togglemesh.NewClient(options)
	if err != nil {
		panic(fmt.Errorf("failed to initialize ToggleMesh client: %v", err))
	}
	defer client.Close() // Ensure graceful shutdown

	fmt.Println("Successfully connected to ToggleMesh!")
    // ...
}
```

### 2. Evaluate Feature Flags
Evaluate flags in memory instantly using `IsEnabled`. You can pass user context properties to match targeting rules (e.g., targeting a specific `Email` domain).

```go
    // Using the constant generated by the CLI
    flagKey := togglemesh.Gmail20percent // generated constant
    
    uid := "user_12345"
    context := map[string]any{
        "Email": "test_user@gmail.com",
        "Plan": "Pro",
    }

    // IsEnabled evaluates instantly in memory
    enabled := client.IsEnabled(flagKey, false, uid, context)
    
    if enabled {
        fmt.Println("New Feature Enabled!")
    } else {
        fmt.Println("Default Flow.")
    }
```

## 📊 Analytics and Events

To track metrics, goal conversions, or Multi-Armed Bandit experiment outcomes, use the `TrackEvent` method. Events are aggregated locally and flushed to the server automatically.

```go
// Track that a user completed a specific action (e.g., played a video)
client.TrackEvent("video_played", 1.0, uid, context)
```

## ⚙️ How it works

1. **Bootstrap**: Upon initialization, the SDK fetches the entire rule set for your environment via REST API.
2. **Stream**: The SDK opens a persistent Server-Sent Events (SSE) connection to listen for flag/rule changes in real time.
3. **Evaluate**: When you call `client.IsEnabled()`, the SDK evaluates the user context against the local, in-memory cache using a custom rule engine (Fractional Rollout, Targeting Rules).
4. **Analytics**: Evaluation metrics and tracked events are buffered locally and flushed asynchronously to avoid blocking the main thread.
